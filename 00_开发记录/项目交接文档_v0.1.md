## 二、数据库架构设计

### 2.1 数据库选型

| 数据库类型 | 选型 | 用途说明 |
|-----------|------|---------|
| **主数据库** | PostgreSQL 15+ | 关系型数据存储（用户、卡牌、战斗记录） |
| **缓存数据库** | Redis 7.0+ | 会话缓存、实时排行榜、匹配队列 |
| **时序数据库** | TimescaleDB | 战斗日志分析、运营数据统计（可选） |
| **对象存储** | MinIO / 阿里OSS | 美术资源、玩家头像等文件存储 |

### 2.2 核心数据表设计

```sql
-- 用户账户表
CREATE TABLE users (
    id              BIGSERIAL PRIMARY KEY,
    username        VARCHAR(50) UNIQUE NOT NULL,
    email           VARCHAR(255) UNIQUE NOT NULL,
    password_hash   VARCHAR(255) NOT NULL,
    level           INT DEFAULT 1,
    experience      BIGINT DEFAULT 0,
    coins           BIGINT DEFAULT 0,
    gems            INT DEFAULT 0,
    created_at      TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at      TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    last_login_at   TIMESTAMP
);

-- 卡牌定义表（模板表）
CREATE TABLE card_templates (
    id              BIGSERIAL PRIMARY KEY,
    card_code       VARCHAR(20) UNIQUE NOT NULL,  -- 例如: FIRE_001
    card_name       VARCHAR(100) NOT NULL,
    element         VARCHAR(20) NOT NULL,          -- fire/water/wind/wood
    rarity          INT NOT NULL,                  -- 1-5星
    cost            INT NOT NULL,                  -- 能量消耗
    attack          INT DEFAULT 0,
    health          INT DEFAULT 0,
    skill_ids       JSONB,                          -- 技能ID数组
    description     TEXT,
    artwork_url     VARCHAR(500),
    created_at      TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 玩家卡牌收藏表
CREATE TABLE user_cards (
    id              BIGSERIAL PRIMARY KEY,
    user_id         BIGINT REFERENCES users(id),
    card_template_id BIGINT REFERENCES card_templates(id),
    card_instance_id BIGINT UNIQUE,                -- 具体卡牌实例ID（用于区分同名卡）
    level           INT DEFAULT 1,
    exp             INT DEFAULT 0,
    is_locked       BOOLEAN DEFAULT FALSE,
    obtained_at     TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 战斗记录表
CREATE TABLE battle_records (
    id              BIGSERIAL PRIMARY KEY,
    user_id         BIGINT REFERENCES users(id),
    opponent_id     BIGINT,
    battle_type     VARCHAR(20),                   -- rank/practice/party
    result          VARCHAR(10),                   -- win/lose/draw
    my_deck         JSONB,
    opponent_deck   JSONB,
    battle_duration INT,                           -- 秒
    mmr_before      INT,
    mmr_after       INT,
    created_at      TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 天梯排名表
CREATE TABLE leaderboard (
    user_id         BIGINT PRIMARY KEY REFERENCES users(id),
    rank            INT,
    mmr             INT DEFAULT 1000,
    tier            VARCHAR(50),                   -- 白银/黄金/钻石等
    stars           INT DEFAULT 0,
    season_id       INT,
    updated_at      TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

### 2.3 Redis 数据结构设计

```
# 用户会话缓存
user:session:{user_id}           -> Hash (token, expire_time)

# 实时排行榜 (Sorted Set)
rank:global                       -> ZSET (user_id -> mmr)

# 匹配队列 (List)
queue:rank:{element_tier}         -> List (user_id)
queue:practice                     -> List (user_id)

# 战斗房间 (Hash)
battle:room:{room_id}             -> Hash (player1, player2, state, start_time)

# 玩家在线状态
user:online:{user_id}            -> String (last_ping_timestamp)

# 每日签到缓存
user:checkin:{user_id}:{date}    -> String "1"
```

## 三、AI生图质量控制方案

### 3.1 统一风格定义

**卡牌艺术风格指南**:

| 属性 | 规范 |
|------|------|
| **画风** | 奇幻卡牌风格，类似游戏王/万智牌 |
| **视角** | 角色居中，45度俯视角度 |
| **构图** | 主体占画面60%，留白边框20% |
| **光效** | 元素对应光效（火红/水蓝/风绿/木棕） |
| **背景** | 渐变暗色+元素粒子效果 |

### 3.2 标准提示词模板

```python
# 基础风格模板
STYLE_TEMPLATE = """
masterpiece, best quality, highly detailed,
[CHARACTER_TYPE] character card illustration,
fantasy trading card art style,
dramatic lighting, [ELEMENT_COLOR] elemental effects,
dark gradient background with magical particles,
centered composition, [ASPECT_RATIO],
professional card game artwork,
Cohesive art style across all cards
"""

# 负面提示词模板（固定不变）
NEGATIVE_PROMPT = """
blurry, low quality, watermark, text, signature,
deformed, bad anatomy, extra limbs, malformed hands,
inconsistent art style, color bleeding,
amateurish, stock photo, 3D render look,
anime style, photorealistic
"""

# 元素特定颜色
ELEMENT_COLORS = {
    "fire": "orange and red flames, heat distortion",
    "water": "blue and cyan waves, water droplets",
    "wind": "green and teal swirls, leaves",
    "wood": "brown and green tones, vines and roots"
}

# 角色类型关键词
CHARACTER_TYPES = {
    "warrior": "armored warrior with sword and shield",
    "mage": "robed wizard with magical staff",
    "archer": "agile elf with bow and arrows",
    "guardian": "massive golem with stone fist"
}
```

### 3.3 质量验证流程

```
AI生成 → 初步筛选(关键词匹配) → 人工审核(评分≥8/10) → 素材库归档
                         ↓
                  不合格 → 调整关键词 → 重新生成
```

### 3.4 批量生成配置

| 参数 | 推荐值 | 说明 |
|------|--------|------|
| 模型 | Illustrious / Flux.2 Dev | 高细节+一致性 |
| 采样步数 | 25-30 | 质量与速度平衡 |
| CFG Scale | 7-8 | 不要太高避免过饱和 |
| 分辨率 | 1024x1024 或 704x1024 | 便于后期裁剪 |
| 每批数量 | 4-8张 | 人工审核效率 |
